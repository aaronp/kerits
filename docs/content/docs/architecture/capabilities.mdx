### keri-said-derivation — Compute and verify KERI-compliant SAIDs for versioned artifacts such as KEL events, TEL events, and ACDC-related artifacts.

**Layers:** core
**Status:** PARTIAL (7 passed, 0 failed)

**Invariants**

- `said-field-binding` — Each artifact family declares the authoritative SAID field used for derivation and verification. Preimage construction replaces exactly that field with the required placeholder, sealing writes the derived SAID back to that same field, and verification recomputes against that same location.
  - ✅ `derives-said-with-placeholder`
  - 📝 `matches-keri-reference-vector-kel-icp`
- `placeholder-at-said-field` — The preimage used to derive a SAID contains the KERI-required placeholder at the SAID field, so digest input is independent of any previously populated value at that field.
  - ✅ `derives-said-with-placeholder`
  - 📝 `matches-keri-reference-vector-kel-icp`
- `derivation-surface-isolates-signed-fields` — Only fields defined by the artifact family's derivation surface contribute to the SAID preimage; excluded fields do not affect the digest.
  - ✅ `excluded-fields-do-not-affect-said`
  - ✅ `recomputes-equal-for-sealed-artifact`
  - ✅ `ignores-tampered-excluded-field`
- `version-string-size-converges` — For artifact families with a version string carrying encoded size, derivation converges on a final serialized form whose encoded size matches the size declared in the sealed artifact.
  - ✅ `version-string-converges-on-serialized-size`
  - ✅ `version-string-converges-for-larger-artifact`
- `sealed-artifact-verifies-against-preimage` — A sealed artifact verifies only when recomputation from its declared derivation surface yields the same SAID; mutation of any included field invalidates verification.
  - ✅ `recomputes-equal-for-sealed-artifact`
  - ✅ `rejects-tampered-included-field`
- `keri-canonical-serialization` — SAID derivation uses the canonical serialization required by KERI for the artifact family, so equivalent artifacts derive the same SAID as other KERI-compliant implementations.
  - 📝 `matches-keri-reference-vector-kel-icp`
  - 📝 `matches-keri-reference-vector-tel-vcp`
